fare-matcher

I (Faré) am trying to experiment on pattern-matching with first-class patterns.

In the old Tunes code, I tried to show how I'd envision a declarative description of a machine's instruction set being used as the source for a assembler, disassembler, and more, based on manipulating first-class patterns. I'd like to experiment in that direction. That's for the goal end of it.

For the code end, I've fiddled a simple pattern in Common Lisp (see http://www.cliki.net/fare-matcher), that could be extended to experiment in that direction.

If a lisp-aware tuneser wants to help, he'll be welcome. Contact me about it.

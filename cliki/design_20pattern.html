Design Pattern

A term for describing how a solution framework is often not supported by underlying programming languages. It became infamous when the "Design Patterns" book for programming emerged, but primarily existed to illustrate how to work around C++, rather than focus on the issues. A later book for Smalltalk was made in contrast. See also pattern languages.

  • Peter Norvig's Design Patterns in Dynamic Programming.
  • Greg Sullivan's follow-up, with actual code to illustrate, using a cooked-up object system in Scheme.
